# MoveValue 서비스 모델

## 한 줄 정의

MoveValue는 시민의 주거 선택을 `월 주거비 + 이동시간 + 교통 접근성 + 생활 인프라 + 안전환경 + 계약 전 위험 신호` 기준으로 재계산해, 실제 생활비와 이동 부담을 줄이고 계약 판단까지 돕는 국토교통 데이터 기반 생활권 매칭 서비스다.

## 문제

주거 의사결정은 월세나 매매가 중심으로 이뤄지지만 실제 시민이 체감하는 비용은 통근시간, 환승 스트레스, 교통비, 생활시설 접근성까지 포함된다. 부동산 플랫폼은 매물 탐색에 강하지만 이동·생활권 비용을 설명 가능한 데이터로 보여주는 기능이 약하고, 지자체는 어떤 생활권의 교통·주거 부담이 큰지 시민 관점으로 보기 어렵다.

## 핵심 고객

- 공공: 지자체, 국토교통 정책 담당 부서, 공공 데이터 활용 사업
- 민간: 부동산 플랫폼, 직장인·청년 대상 주거 서비스, MaaS·교통 플랫폼
- 시민: 청년 1인 가구, 신혼부부, 출퇴근 직장인, 교통약자 가족

## 핵심 기능

1. 예산·출퇴근 목적지·가구 유형을 입력받는다.
2. 후보 생활권별 주거비, 통근시간, 환승 접근성, 생활 SOC, 안전·환경 접근성을 계산한다.
3. 실제 전월세 예시와 점수 산출 근거를 항목별로 공개해 사용자가 납득할 수 있게 한다.
4. 집 주소와 회사 주소를 입력해 ODsay/TMAP live 경로 또는 폴백 경로로 통근 루트를 검증한다.
5. 지도에서 추천 생활권 주변의 서울 아파트 단지 위치와 세대수·동수·사용승인일을 확인하고, 라벨을 매매가·전세가율·위험도·통근시간으로 전환한다.
6. 선택 단지의 가격 API 연계 상태, 거래 추이, 전세 위험 신호, 계약 전 체크리스트, 생활 SOC 반경, AI 요약을 대시보드로 확인한다.
7. 지자체·플랫폼에는 생활권 수요 리포트와 API를 제공한다.
8. 데이터 제공자는 오픈마켓 또는 협약 형태로 데이터 판매·공급 채널에 참여한다.

## 밸류체인

| 단계 | 참여자 | 가치 |
| --- | --- | --- |
| 데이터 공급 | 국토교통부, 공공데이터포털, VWorld, 철도·교통 데이터 기관, 민간 이동 데이터 사업자 | 원천 데이터 제공과 유통 |
| 데이터 가공 | MoveValue | 주소 정규화, 좌표화, 생활권 단위 결합, 점수화 |
| 서비스 운영 | MoveValue 웹/API | 시민 추천 화면, 플랫폼 API, 정책 리포트 |
| 수요처 | 시민, 지자체, 부동산·교통 플랫폼 | 비용 절감, 의사결정 개선, 정책 타깃팅 |
| 환류 | 사용자 피드백, 조회·선호 데이터 | 지역별 수요와 데이터 품질 개선 |

## 수익 모델

- B2C: 개인용 상세 리포트, 알림, 후보지 비교 프리미엄
- B2B: 부동산 플랫폼 API 과금, 기업 복지·채용 이주 지원 패키지
- B2G: 지자체 생활권 진단 리포트, 교통·주거 취약지 분석 과제
- 데이터 마켓 연계: 민간 교통·유동 데이터 구매 수요 창출 및 분석 상품화

## 차별성

- 부동산 가격이 아니라 `총 생활 이동비용`과 안전환경까지 핵심 지표로 본다.
- 네이버부동산·다방·직방처럼 매물을 먼저 나열하는 방식이 아니라, 생활권을 먼저 추천하고 그 안의 실제 전월세 예시와 통근 루트를 검증한다.
- 추천 결과마다 데이터 근거와 항목별 점수를 보여주고, 단지 상세에서는 가격 근거·통근 근거·위험 근거·생활권 근거·확인 필요 서류를 분리한다.
- 공공데이터만으로 최소 기능을 구현하고, 민간 이동 데이터가 붙을수록 정밀도가 올라가는 단계형 구조다.
- 시민 서비스, 플랫폼 API, 지자체 정책 리포트로 동시에 확장 가능한 밸류체인 모델이다.

## 구현 범위

현재 프로토타입은 API 기반 웹 서비스로 다음을 구현했다.

- 서울시 2025 전월세 실데이터 기반 생활권 주거비 집계와 후보 전월세 예시
- Python API 서버 기반 추천 점수 계산
- 예산, 목적지, 가구 유형, 우선순위 조정
- 후보 생활권 랭킹("전체 9개 중 상위 n개")과 항목별 점수, 카드별 숫자 기반 추천 근거
- 독립 `통근검증` 화면: 검증 생활권을 바꾸면서 Kakao 주소 검색, ODsay/TMAP 대중교통 경로 API, 거리 기반 폴백을 결합한 집-회사 통근 루트 확인
- Leaflet + OpenStreetMap 실제 지도: 점수별 색상·크기 생활권 마커 9개, 서울시 공동주택 아파트 단지 레이어, 매매가·전세가율·위험도·통근시간 라벨 필터, 선택 단지 생활 SOC 반경, 카드↔마커 양방향 선택 연동, 팝업(생활권명·통근시간·월세 중앙값·종합점수, 단지명·세대수·동수·사용승인일), 로딩 실패 시 분포도 폴백
- 단지 상세 대시보드: OpenAptInfo 기본 정보, 국토교통부 실거래가 live 보정 상태, 가격·거래 추이, 전세 위험 신호, 계약 전 체크리스트, 단지 비교, AI Agent 질의응답
- 상단 서비스 메뉴(추천·통근검증·지도·데이터 근거·API·사업모델·제출자료)의 탭형 화면 전환
- 데이터 근거 섹션: 생활권별 매칭 거래 건수, 월세·보증금·전세 중앙값, SOC, 안전환경 표와 스냅샷 한계 고지
- API 섹션: 엔드포인트 설명, 호출 예시, B2B/B2G 확장 방향
- 사업모델 섹션: B2C/B2B/B2G/데이터 마켓 수익 채널과 밸류체인
- 제출자료 섹션: 신청서 DOCX·문서 경로 안내

선정 후 구체화 기간에는 실제 API 키 주입 검증 결과 기록, ODsay/TMAP 호출 캐시, 국토교통부 실거래가 단지명 매칭률 개선, 공시가격 PNU 매핑, 건축물대장·용도지역 연계, 서울 열린데이터광장 전체 단지 갱신 배치, 지도 SDK 교체 옵션(VWorld·카카오·네이버), SOC·안전·환경 원천 API 자동 갱신, 데이터 업데이트 배치, 사용자 저장 기능, 정책 리포트 PDF 생성 기능을 추가한다. iOS Swift 앱은 현재 API를 소비하는 별도 클라이언트로 확장한다. 상세 순서는 `docs/product-roadmap.md`에 정리했다.
