# MoveValue 시스템 흐름 요약

## 전체 서비스 아키텍처

MoveValue는 현재 Python 표준 라이브러리 기반 API 서버와 HTML/CSS/Vanilla JS 웹앱으로 구성된다. 프론트엔드는 지도, 조건 입력, 상세 패널, 비교 패널을 담당하고, 서버는 추천 점수 계산, 통근 경로 어댑터, 부동산 상세 모델, AI Agent 응답, 정적 파일 제공을 담당한다.

## 사용자 입력 흐름

1. 사용자가 월 주거 예산, 회사/목적지 주소, 가구 유형, 점수 가중치를 입력한다.
2. 프론트엔드는 `/api/apartment-recommendations`에 조건을 전달한다.
3. 서버는 아파트 단지별로 가까운 생활권, 통근시간, 주거비, 생활 SOC, 안전 점수를 계산한다.
4. 결과는 추천 카드와 지도 마커에 동시에 반영된다.

## 지도 기반 추천 흐름

1. 지도 초기화 시 `/api/apartments`가 단지/클러스터 feature를 반환한다.
2. 지도 bounds와 zoom에 따라 서버가 클러스터링 여부를 결정한다.
3. 프론트엔드는 가격, 전세가율, 위험도, 통근시간 라벨을 지도 마커에 표시한다.
4. 추천 카드를 누르면 지도 중심과 상세 패널이 선택 단지로 이동한다.

## 부동산 상세 대시보드 흐름

1. 사용자가 지도 마커 또는 추천 카드를 선택한다.
2. 프론트엔드는 `/api/property-detail?id=...`을 호출한다.
3. 서버는 단지 기본정보, 생활권 데이터, 가격 추정/live 보정, 위험 신호, AI 요약을 결합한다.
4. 화면은 기본 정보, 가격 정보, 거래 추이, 전세 위험 신호, 계약 전 체크리스트, AI Agent를 표시한다.

## 전세 위험 신호 점검 흐름

1. 매매가, 전세가, 공시가격, 주변 전세 중앙값, 가격 변동성, 건축연도, 등기부 입력 여부를 결합한다.
2. 전세가율, 공시가격 대비 보증금, 주변 시세 대비 차이, 최근 변동성, 노후도, 권리관계 미입력 여부를 신호화한다.
3. 결과는 법적 판정이 아니라 `위험 신호 점검`, `계약 전 확인 필요`, `주의 요소`로만 표현한다.

## AI Agent 질의응답 흐름

1. 사용자가 상세 대시보드에서 질문을 입력한다.
2. `/api/property-agent`가 해당 단지 상세 데이터와 비교 후보를 로드한다.
3. 규칙 기반 Agent가 질문 의도를 전세 안전성, 추천 이유, 안전한 대안 탐색으로 분류한다.
4. 답변은 가격 근거, 통근 근거, 위험 근거, 주변 입지 근거, 확인 필요 서류로 나뉘어 반환된다.

## 데이터 수집·정제·가공·점수화 흐름

| 단계 | 처리 | 산출 |
| --- | --- | --- |
| 수집 | 서울시 전월세, 서울 OpenAptInfo, SOC/안전 좌표, 경로/가격 API | 원천 파일 또는 live 응답 |
| 정제 | 면적, 금액, 법정동, 좌표, 대표역 기준 통일 | 정규화 row |
| 집계 | 생활권 반경, 법정동, 단지 단위 집계 | `areas.actual.json`, apartment feature |
| 점수화 | 통근, 주거비, SOC, 안전, 위험 신호 가중합 | 추천 점수, 위험 등급 |
| 설명화 | 수치 기반 추천 이유와 Agent 근거 그룹 생성 | 사용자 설명 문장 |

## 외부 API 연동 구조

| API | 현재 구조 | 키 |
| --- | --- | --- |
| Kakao Local | 상세 주소를 좌표로 변환 | `KAKAO_REST_API_KEY` |
| ODsay | 대중교통 경로 계산 | `ODSAY_API_KEY` |
| TMAP Transit | 대중교통 경로 계산 | `TMAP_APP_KEY` |
| 서울 OpenAptInfo | 서울 공동주택 단지 전체 조회 | `SEOUL_OPEN_API_KEY` |
| 국토교통부 실거래가 | 아파트 매매/전월세 live 보정 | `MOLIT_SERVICE_KEY` 등 |
| 공동주택 공시가격 | 공시가격 정밀화 | `PUBLIC_PRICE_API_KEY` 등 |
| VWorld | 주소/좌표/용도지역 확장 | `VWORLD_API_KEY` |
| 건축HUB | 건축물대장 속성 확장 | `BUILDING_HUB_API_KEY` |

## 프론트엔드와 데이터 레이어 연결 구조

프론트엔드는 서버 API를 통해서만 데이터를 읽는다. 서버는 로컬 JSON과 외부 API 어댑터를 같은 응답 포맷으로 감싸기 때문에, 키가 없을 때도 화면은 깨지지 않고 폴백 상태를 표시한다.

## 확장 가능한 모듈 구조

| 확장 방향 | 현재 연결점 |
| --- | --- |
| iOS SwiftUI 앱 | 기존 `/api/*` 계약 재사용 |
| PostgreSQL/PostGIS | `data/*.json`을 공간 테이블로 이전 |
| live API 캐싱 | `route_adapters.py`, `real_estate_price_adapters.py`에 캐시 레이어 추가 |
| LLM Agent | `property_agent_answer()`를 RAG/LLM 계층으로 교체 |
| B2G 리포트 | 생활권 점수와 위험 신호를 행정동 단위로 집계 |
