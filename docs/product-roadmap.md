# MoveValue 제품 로드맵

공모전 프로토타입(현재) 이후 선정·구체화 단계에서 진행할 개발 순서를 정리한다. 원칙은 "실데이터 어댑터를 하나씩 추가할수록 정밀도가 올라가는 단계형 구조"를 유지하는 것이다.

## 0단계 - 현재 완료 상태

- 서울시 2025 전월세 실데이터 기반 생활권 9개 주거비 집계 (114,319건 매칭)
- Python 표준 라이브러리 API 서버 + Leaflet/OSM 실제 지도 웹 서비스
- 추천 랭킹·한 줄 근거·상세 점수·데이터 근거 표·사업모델·제출자료 화면
- 통근시간은 대중교통 경로 API 어댑터 구현 완료, API 키 미설정 시 MVP 테이블 폴백
- 사용자가 입력한 집·회사 위치의 통근 루트 검증 UI 및 `/api/commute-route` 구현 완료
- 생활 SOC는 병원·학교·공원 좌표 스냅샷 반경 집계 기반으로 교체 완료
- 안전·환경은 치안시설·CCTV 집계점·도시대기 측정망·공원 접근성 스냅샷 기반으로 교체 완료
- 생활권별 전월세 실거래 예시 4건과 숫자 기반 추천 사유 문장 구현 완료
- 경쟁 서비스 대비 포지션: 매물 검색이 아니라 주거비·통근·생활 SOC·안전환경 통합 생활권 추천/API 엔진

## 1단계 - 데이터 정밀도 (선정 직후 1~2개월)

1. **대중교통 경로 API 운영 고도화**: ODsay/TMAP 어댑터의 API 키 운영, 호출 캐시, 실패 로그, 혼잡도, 첫차·막차, GTFS 라우팅 비교 지원.
2. **생활 SOC 실데이터 자동 갱신**: 현재 병원·학교·공원 좌표 스냅샷을 서울 열린데이터광장 API 기반 전체 자동 갱신으로 확장하고, 편의시설 카테고리를 추가.
3. **안전·환경 원천 API 자동 갱신**: 현재 스냅샷 점수를 CCTV 원천, 치안시설, 범죄안전지수, 대기질, 녹지율 공공데이터 전체 커버리지로 확장.
4. **주거비 월별 증분 갱신 배치**: 서울시 전월세 파일 자동 수집 + 국토교통부 실거래가 API 병행, 매물 예시 최신성 관리.

## 2단계 - 서비스 확장 (3~4개월)

1. **후보 생활권 확대**: 서울 전체 역세권 → 수도권 → 광역시. 법정동 매핑 자동화.
2. **지도 제공자 옵션**: VWorld 타일 전환(공공 신뢰도), 카카오/네이버 SDK 옵션. `docs/api-development.md`의 교체 가이드 기준.
3. **사용자 기능**: 조건 저장, 후보지 비교(2~3개 나란히), 변동 알림.
4. **iOS Swift 앱**: 기존 `/api/areas`·`/api/recommendations`를 소비하는 네이티브 클라이언트.

## 3단계 - 수익화 인프라 (5~6개월)

1. **B2B API 상품화**: API 키 발급, 사용량 계측, 호출량 과금, 파트너 대시보드.
2. **B2G 리포트 생성기**: 생활권 취약지 진단 PDF 자동 생성, 지자체 실증 과제 템플릿.
3. **데이터 마켓 연계**: 국가교통 데이터 오픈마켓 민간 이동 데이터 구매·결합 파이프라인.

## 항상 유지할 원칙

- 추천 점수는 항목별 근거와 데이터 출처를 함께 공개한다 (설명 가능성).
- 실데이터와 프록시 지표를 화면에서 구분 표기한다 (정직한 한계 고지).
- 공공데이터만으로도 기본 기능이 동작해야 한다 (개방형 최소 구성).
