데이터 출처

정직한 한계
통근시간은 대중교통 경로 API 어댑터를 먼저 사용하되 API 키가 없으면 MVP 테이블로 폴백합니다. 안전·환경 점수는 MVP 프록시이며, 선정 후 안전·대기·녹지 공공데이터 어댑터로 고도화합니다.

API

동일 엔진을 외부 파트너에게 제공

웹 화면이 사용하는 추천 엔진을 그대로 JSON API로 노출합니다. 외부 패키지 없이 Python 표준 라이브러리만으로 실행되어 심사 환경에서 재현성이 높습니다.

GET/api/health

데이터 파일 로딩 상태, 후보 생활권 수, 데이터 출처 메타를 반환합니다. 운영 모니터링과 파트너 연동 사전 점검에 사용합니다.

GET/api/areas

정규화된 생활권 후보 전체와 실데이터 집계 근거(매칭 거래 건수, 법정동, 출처)를 반환합니다. 지도 초기화와 데이터 근거 표가 이 응답을 사용합니다.

GET/api/recommendations

예산·목적지·가구 유형·우선순위 가중치를 받아 설명 가능한 점수와 함께 추천 랭킹을 반환합니다. 항목별 점수가 포함되어 "왜 추천됐는지" 추적할 수 있습니다.

추천 API 호출 예시 
curl 'http://127.0.0.1:5173/api/recommendations?budget=70&destination=gangnam&persona=single&commuteWeight=35&costWeight=30&serviceWeight=20&safetyWeight=15&limit=9'

B2B·B2G 확장
부동산·교통 플랫폼은 이 API를 매물 상세 화면의 생활권 점수 위젯으로 임베드할 수 있고, 지자체는 동일 엔진으로 생활권별 주거-이동 부담 리포트를 배치 생성할 수 있습니다. 인증 키 발급·사용량 과금 계층은 로드맵에 포함되어 있습니다.

사업모델

하나의 엔진, 세 개의 수익 채널
B2C 개인 생활권 리포트

예산·통근 조건 기반 후보지 비교, 상세 리포트, 조건 변동 알림을 프리미엄으로 제공합니다.

B2B 플랫폼·기업 API

부동산 플랫폼 생활권 점수 위젯, 기업 채용·이주지원 패키지에 추천 API를 호출량 기반으로 과금합니다.

B2G 지자체 취약지 리포트

생활권별 주거-이동 부담을 시민 관점으로 진단해 교통·주거 정책 타깃팅 리포트로 제공합니다.

데이터 마켓 교통·생활 데이터 수요 창출

국가교통 데이터 오픈마켓의 민간 이동 데이터를 구매·결합해 정밀도를 높이고, 분석 상품으로 재공급합니다.

서비스 밸류체인

① 데이터 공급국토교통부·서울시 실거래/전월세, 역사 좌표, 교통·생활 SOC 공공데이터, 민간 이동 데이터
② MoveValue 가공주소 정규화, 생활권 단위 결합, 설명 가능한 점수화 — 데이터 산업의 가공·부가가치 단계
③ 서비스/API시민 추천 화면, 플랫폼 점수 API, 지자체 정책 리포트 — 국민 체감 가치로 전환
④ 수요·환류시민 피드백, 지자체 실증, 데이터 구매 수요가 다시 데이터 생태계로 환류

국토교통 데이터 생태계 기여
공공데이터만으로 최소 기능을 구현하고, 민간 데이터가 결합될수록 정밀도가 올라가는 단계형 구조입니다. 데이터 공급자에게는 새로운 수요처를, 국민에게는 체감 가능한 주거-이동 의사결정 가치를 만듭니다.
