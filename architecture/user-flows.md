# User Flows

이 문서는 MoveValue의 핵심 사용자 흐름을 발표와 QA 기준으로 정리한다. 현재 구현은 웹 대시보드 중심이며, 같은 API를 iOS 앱에서 재사용할 수 있게 설계했다.

## Flow 1. 생활권 추천

```mermaid
flowchart TD
    A["사용자 진입<br/>추천 탭"] --> B["예산 입력"]
    B --> C["목적지 선택<br/>강남·여의도·서울역·디지털·판교"]
    C --> D["가구 유형 선택<br/>1인·직장인·신혼·시니어"]
    D --> E["가중치 조정<br/>통근·주거비·SOC·안전환경"]
    E --> F["추천 계산 요청<br/>/api/recommendations"]
    F --> G["생활권 TOP 랭킹 표시"]
    G --> H["카드 선택"]
    H --> I["상세 근거 표시<br/>실거래 예시·SOC·안전환경"]
    H --> J["지도 마커와 동기화"]
    I --> K["통근검증 또는 지도 화면으로 이동"]
```

### 성공 기준

- 추천 카드에 추상 문구가 아니라 통근시간, 월세, 병원·학교·공원, 치안시설·CCTV 수치가 표시된다.
- 카드 우측 점수 중심 UI보다 순위와 근거 문장이 먼저 보인다.
- API 키가 없어도 추천 결과가 렌더링된다.

## Flow 2. 통근 루트 검증

```mermaid
flowchart TD
    A["통근검증 탭"] --> B["추천 생활권 선택"]
    B --> C["집 주소 자동 입력<br/>대표 주소"]
    C --> D["회사 주소 입력"]
    D --> E["경로 API 선택<br/>자동·ODsay·TMAP"]
    E --> F["/api/commute-route 호출"]
    F --> G{"live API 가능?"}
    G -->|"가능"| H["실제 대중교통 경로<br/>시간·환승·도보·요금"]
    G -->|"불가"| I["거리 기반 폴백<br/>추정값과 제한 사유 표시"]
    H --> J["지도 경로선 표시"]
    I --> J
```

### 성공 기준

- 기본 집 위치는 좌표가 아니라 실제 주소로 보인다.
- Kakao/ODsay/TMAP 키는 환경변수로만 사용한다.
- live 경로가 불가능하면 실패 화면 대신 폴백 결과와 제한 사유를 표시한다.

## Flow 3. 지도 기반 부동산 대시보드

```mermaid
flowchart LR
    A["지도 탭"] --> B["풀스크린 지도 대시보드"]
    B --> C["왼쪽 사이드바<br/>조건·추천 TOP5·단지 리스트"]
    B --> D["오른쪽 지도<br/>생활권 마커·단지 가격 라벨"]
    D --> E["생활권 마커 클릭"]
    E --> F["추천 카드/상세 근거 동기화"]
    D --> G["아파트 단지 마커 클릭"]
    G --> H["부동산 상세 드로어 열기"]
    H --> I["가격·거래 추이·위험 신호·생활권·AI 요약"]
```

### 성공 기준

- 지도는 카드 안 작은 이미지가 아니라 참고 부동산 서비스처럼 화면 중심이 된다.
- 단지 마커는 가격 미리보기와 위험 신호 톤을 보여준다.
- 단지 클릭 시 상세 드로어가 열리고 지도 탐색 맥락이 유지된다.

## Flow 4. 단지 비교

```mermaid
flowchart TD
    A["단지 상세 드로어"] --> B["비교에 추가"]
    B --> C{"비교 후보 수"}
    C -->|"1개"| D["현재 단지 기준 정보 표시"]
    C -->|"2~3개"| E["가격·전세가율·위험 등급·생활권 점수 비교"]
    E --> F["더 안전하거나 비용 대비 효율 높은 후보 판단"]
    F --> G["비교 후보 제거 또는 다른 단지 선택"]
```

### 성공 기준

- 후보는 최대 3개까지 유지해 모바일 화면에서도 비교가 무너지지 않는다.
- 비교표는 단순 시세가 아니라 전세 위험 신호와 생활권 정보를 함께 보여준다.

## Flow 5. AI Agent 질의응답

```mermaid
sequenceDiagram
    participant User as 사용자
    participant UI as 상세 드로어
    participant API as /api/property-agent
    participant Model as property_model.py

    User->>UI: "이 아파트 전세 들어가도 괜찮아?"
    UI->>API: id + question
    API->>Model: 상세 데이터와 비교 후보 로드
    Model->>Model: 질문 의도 분류<br/>전세 안전성·추천 이유·대안 탐색
    Model-->>API: answer + basis + disclaimer
    API-->>UI: 근거 기반 응답
    UI-->>User: 장점, 주의사항, 계약 전 확인 항목 표시
```

### 답변 원칙

- 법적 판정처럼 말하지 않는다.
- `위험 신호 점검`, `계약 전 확인 필요`, `주의 요소` 같은 표현을 사용한다.
- 가격, 통근, 생활 SOC, 안전환경, 주변 비교를 근거로 함께 제시한다.

## Flow 6. 정책·B2G 리포트 확장

```mermaid
flowchart TD
    A["지자체·공공기관"] --> B["관심 생활권 또는 행정동 선택"]
    B --> C["청년·신혼·고령자 등 대상 조건 선택"]
    C --> D["MoveValue API 호출"]
    D --> E["생활권별 주거비·통근·SOC·안전환경 격차 분석"]
    E --> F["정책 우선순위 리포트<br/>교통 개선·SOC 보강·전세 위험 모니터링"]
```

### 차별점

네이버부동산, 직방, 다방이 매물 검색 중심이라면 MoveValue는 생활권 추천과 전세 위험 신호를 API·정책 리포트로 확장할 수 있다.
